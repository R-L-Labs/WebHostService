  <name>Task 1: Optimize getInquiries query pattern</name>
  <files>server/src/controllers/inquiries.controller.js</files>
  <action>
    Current pattern (3 queries):
    ```javascript
    const [inquiries, totalCount] = await Promise.all([
      prisma.inquiry.findMany({ ... }),
      prisma.inquiry.count({ where }),
    ]);
    const newInquiriesCount = await prisma.inquiry.count({ where: { status: 'NEW' } });
    ```

    Optimized pattern (2 queries):
    ```javascript
    const [inquiries, totalCount, newInquiriesCount] = await Promise.all([
      prisma.inquiry.findMany({ ... }),
      prisma.inquiry.count({ where }),
      prisma.inquiry.count({ where: { status: 'NEW' } }),
    ]);
    ```

    Move the newInquiriesCount query into the existing Promise.all to run all 3 queries concurrently instead of 2 concurrent + 1 sequential.

    This reduces total query time from (max(query1, query2) + query3) to max(query1, query2, query3).
  </action>
  <verify>
    Server starts without errors.
    GET /api/inquiries returns correct data with newInquiriesCount.
  </verify>
  <done>getInquiries runs all database queries concurrently. No sequential query after Promise.all.</done>
</task>

      id                String        @id @default(uuid())
