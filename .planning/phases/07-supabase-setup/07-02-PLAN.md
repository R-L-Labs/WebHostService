**Database tables (from Prisma schema):**
- users - Admin users (ADMIN, SUPER_ADMIN roles)
- clients - Business clients
- payments - Payment records (linked to clients)
- inquiries - Contact form submissions
- packages - Service packages

**Current authorization pattern:**
- All endpoints except public inquiry submission require JWT auth
- Any authenticated user can access all data (no row-level filtering by user)
- SUPER_ADMIN can manage users, ADMIN cannot

**Key insight:**
This is an admin-only app. All authenticated users are admins who can see all data.
RLS just needs to ensure: authenticated = full access, anonymous = no access.
Exception: inquiries table needs public INSERT for contact form.
</context>

<task type="auto">
  <name>Task 1: Enable RLS and create base policies</name>
  <files>supabase-rls-policies.sql (to execute in Supabase SQL editor)</files>
  <action>
    Create SQL file with RLS policies. The user will run this in Supabase SQL Editor.
    
    Pattern for admin-only access:
    ```sql
    -- Enable RLS
    ALTER TABLE table_name ENABLE ROW LEVEL SECURITY;
    
    -- Authenticated users can do everything
    CREATE POLICY "Authenticated users have full access"
    ON table_name
    FOR ALL
    TO authenticated
    USING (true)
    WITH CHECK (true);
    ```
    
    Special case for inquiries (public can INSERT):
    ```sql
    -- Public can submit inquiries
    CREATE POLICY "Anyone can submit inquiries"
    ON inquiries
    FOR INSERT
    TO anon
    WITH CHECK (true);
    ```
    
    Tables to configure:
    1. users - authenticated only (all operations)
    2. clients - authenticated only (all operations)  
    3. payments - authenticated only (all operations)
    4. inquiries - authenticated for all, PLUS anon for INSERT
    5. packages - authenticated for write, public for read (packages shown on public site)
    
    Create the SQL file at: .planning/phases/07-supabase-setup/rls-policies.sql
  </action>
