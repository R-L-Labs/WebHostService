**Current patterns:**
- Pages import from `../../utils/api` and call `clientsAPI.getAll()`, etc.
- Results accessed via `response.data.data.{property}`
- Stores are updated via setter methods

**Target patterns:**
- Pages import from `@/lib/queries` (or `../../lib/queries`)
- Results accessed directly: `{ clients, total, error }`
- Same store update patterns, just different data source

**Query function signatures (from Phase 9):**
```javascript
// clients.js
getClients({ page, limit, search, status }) → { clients, total, error }
getClient(id) → { client, error }
createClient(data) → { client, error }
updateClient(id, data) → { client, error }
deleteClient(id) → { error }
getClientStats() → { stats: { totalClients, activeClients, prospectClients }, error }

// payments.js
getPaymentsByClient(clientId) → { payments, error }
createPayment(data) → { payment, error }
updatePayment(id, data) → { payment, error }
deletePayment(id) → { error }

// inquiries.js
getInquiries({ page, limit, status }) → { inquiries, total, error }
```

    2. Update fetchData function:
       ```javascript
       const fetchData = async () => {
         try {
           const [statsResult, inquiriesResult] = await Promise.all([
             getClientStats(),
             getInquiries({ limit: 5, page: 1 }),
           ]);

           if (statsResult.error) throw statsResult.error;
           if (inquiriesResult.error) throw inquiriesResult.error;

           setStats(statsResult.stats);
           setRecentInquiries(inquiriesResult.inquiries);
         } catch (error) {
           console.error('Error fetching dashboard data:', error);
           toast.error('Failed to load dashboard data');
         } finally {
           setLoading(false);
         }
       };
       ```

    Key changes:
    - No more `response.data.data.stats` - use `statsResult.stats` directly
    - No more `response.data.data.inquiries` - use `inquiriesResult.inquiries`
    - Error handling checks `result.error` before using data
