<task type="auto" tdd="true">
  <name>Task 2: Write tests for jwt.utils.js</name>
  <files>server/src/utils/jwt.utils.test.js</files>
  <test-first>
    Tests for generateToken:
    - Should return a string token when given valid user object
    - Token should contain userId, email, role in payload

    Tests for verifyToken:
    - Should decode a valid token and return payload
    - Should throw error for invalid token
    - Should throw error for expired token (if testable)
  </test-first>
  <action>
    Create jwt.utils.test.js next to jwt.utils.js.

    Setup:
    - Set process.env.JWT_SECRET before tests (use beforeEach)
    - Import generateToken, verifyToken from jwt.utils.js

    Test generateToken:
    - Create mock user: { id: 'uuid', email: 'test@test.com', role: 'ADMIN' }
    - Call generateToken(user)
    - Assert result is string
    - Decode token (without verifying) to check payload contains userId, email, role

    Test verifyToken:
    - Generate a valid token first
    - Call verifyToken on it
    - Assert returned object has userId, email, role

    Test invalid token:
    - Call verifyToken('invalid.token.here')
    - Assert it throws 'Invalid or expired token'

    Use jwt.decode() from jsonwebtoken to inspect token payload without verification.
  </action>
  <verify>cd server && npm test -- jwt.utils.test.js</verify>
  <done>All JWT utility tests pass. Token generation and verification behavior documented in tests.</done>
</task>
