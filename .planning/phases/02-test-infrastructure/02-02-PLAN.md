<task type="auto" tdd="true">
  <name>Task 3: Write tests for auth.controller.js login function</name>
  <files>server/src/controllers/auth.controller.test.js</files>
  <test-first>
    Tests for login:
    - Should return 400 if email missing
    - Should return 400 if password missing
    - Should return 401 if user not found
    - Should return 401 if password incorrect
    - Should return 200 with token and user if credentials valid
  </test-first>
  <action>
    Create auth.controller.test.js.

    Setup:
    - Import login from auth.controller.js
    - Import mockPrisma from test mocks
    - Create mock req, res, next objects
    - Set process.env.JWT_SECRET

    Mock req pattern:
    ```javascript
    const mockReq = { body: { email: 'test@test.com', password: 'password' } };
    const mockRes = {
      status: vi.fn().mockReturnThis(),
      json: vi.fn()
    };
    const mockNext = vi.fn();
    ```

    Test missing email:
    - req.body = { password: 'test' }
    - Call login(mockReq, mockRes, mockNext)
    - Assert res.status called with 400

    Test missing password:
    - req.body = { email: 'test@test.com' }
    - Assert res.status called with 400

    Test user not found:
    - mockPrisma.user.findUnique.mockResolvedValue(null)
    - Assert res.status called with 401

    Test password mismatch:
    - mockPrisma.user.findUnique.mockResolvedValue({ password: 'wronghash' })
    - Assert res.status called with 401

    Test successful login:
    - Hash a real password first
    - mockPrisma.user.findUnique.mockResolvedValue({ id: '1', email: 'test@test.com', password: hashedPassword, role: 'ADMIN' })
    - req.body = { email: 'test@test.com', password: 'realpassword' }
    - Assert res.status called with 200
    - Assert res.json called with object containing token
  </action>
  <verify>cd server && npm test -- auth.controller.test.js</verify>
  <done>Auth controller login tests pass. All validation and auth paths covered.</done>
