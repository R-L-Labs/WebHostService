**Database considerations:**
- All tables have soft deletes (deleted_at column) - filter in queries
- Table names are lowercase: clients, inquiries, packages
- RLS policies require authenticated user for clients/inquiries
- packages are publicly readable

**Supabase query patterns:**
```javascript
// Select with filter
const { data, error } = await supabase
  .from('table')
  .select('*')
  .is('deleted_at', null)

// Insert
const { data, error } = await supabase
  .from('table')
  .insert({ ... })
  .select()
  .single()

// Update
const { data, error } = await supabase
  .from('table')
  .update({ ... })
  .eq('id', id)
  .select()
  .single()

// Soft delete
const { error } = await supabase
  .from('table')
  .update({ deleted_at: new Date().toISOString() })
  .eq('id', id)
```
