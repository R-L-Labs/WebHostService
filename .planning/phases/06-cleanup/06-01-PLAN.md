# Phase 6 Plan 1: Soft Deletes

**Replace hard deletes with soft deletes using deletedAt timestamp**

## Goal

Implement soft deletes for data retention:
- Client, Payment, and Inquiry records are never permanently deleted
- Deleted records are filtered from normal queries
- Data can be recovered if needed

## Pre-Execution Checklist

- [x] Identified hard delete locations: clients, payments, inquiries controllers
- [x] User and Package models excluded (admin data, rarely deleted)
- [x] Prisma supports deletedAt pattern natively

## Tasks

### Task 1: Add deletedAt Field to Schema

**Goal:** Add soft delete column to Client, Payment, and Inquiry models

**Commit Type:** `feat`

Steps:
1. Add `deletedAt DateTime?` field to Client model in schema.prisma
2. Add `deletedAt DateTime?` field to Payment model
3. Add `deletedAt DateTime?` field to Inquiry model
4. Run `npx prisma generate` to update client

**Verification:**
- [ ] Schema validates successfully
- [ ] Prisma client regenerated

### Task 2: Update Controllers to Use Soft Delete

**Goal:** Replace prisma.*.delete with update setting deletedAt

**Commit Type:** `refactor`

Steps:
1. Update `deleteClient` in clients.controller.js:
   ```js
   await prisma.client.update({
     where: { id },
     data: { deletedAt: new Date() },
   });
   ```
2. Update `deletePayment` in payments.controller.js similarly
3. Update `deleteInquiry` in inquiries.controller.js similarly

**Verification:**
- [ ] Delete endpoints set deletedAt instead of removing
- [ ] All tests still pass

### Task 3: Filter Soft-Deleted Records from Queries

**Goal:** Exclude deleted records from normal list/get operations

**Commit Type:** `refactor`

Steps:
1. Update `getClients` to add `where: { deletedAt: null }` condition
2. Update `getClient` to check deletedAt is null
3. Update `getClientPayments` to filter deleted payments
4. Update `getPayment` to check deletedAt is null
5. Update `getInquiries` to add `where: { deletedAt: null }` condition
6. Update `getInquiry` to check deletedAt is null

**Verification:**
- [ ] Soft-deleted records excluded from list views
- [ ] 404 returned for soft-deleted individual records
- [ ] All tests pass

## Completion Checklist
